# 为 Last Kick 贡献

感谢你愿意改进 Last Kick。小修复可以直接提交 Pull Request；如果准备调整玩法规则、视觉方向或引入新的依赖，建议先开 Issue 说明目标和取舍，避免重复劳动。

## 开发环境

- Node.js 24+
- pnpm 11+
- 支持 WebGL 和 Web Audio 的现代浏览器

```bash
pnpm install
pnpm dev
```

## 项目原则

贡献应尽量保持以下边界：

1. **运行时无 AI/API**：AI 可以帮助开发，但公开游戏不应要求 API Key、账号或远程 AI 请求。
2. **判定可解释且确定**：相同输入必须得到相同结果；不要用隐藏随机数调节输赢。
3. **移动端优先**：新交互需要支持触控，并在窄屏和横屏下保持可用。
4. **反馈强于装饰**：动画和声音应强化蓄力、出脚、飞行或结果，不要只增加视觉噪音。
5. **中英文同步**：新增用户可见文案时，同时更新中文和英文。
6. **素材许可可追溯**：优先使用原创、程序化或 CC0 素材；新增第三方素材时记录来源、许可和处理过程。

## 修改玩法规则

射门甜区和判定集中在 `src/experience/store.ts`。界面反馈与最终判定必须继续共用同一份规则，避免出现“提示能进、结果却被扑”的不一致。

修改后运行：

```bash
node scripts/verify-shot-difficulty.mjs
```

如果有意改变当前难度，请同时更新校准断言，并在 Pull Request 中解释目标玩家、预期变化和验证方式。

## 提交前检查

```bash
node scripts/verify-shot-difficulty.mjs
pnpm build
```

另外请手动完成以下检查：

- 在移动端尺寸下完成三脚挑战。
- 分别验证爆射、弧线、勺子的进球与失败反馈。
- 验证门将能够识别重复射法和方向。
- 验证静音、首次声音解锁和降低动态效果模式。
- 切换中英文，确认布局没有溢出。
- 生成一次挑战分享图。

视觉改动请在 Pull Request 中附上修改前后截图或短视频，并注明测试的浏览器和视口尺寸。

## Pull Request 范围

- 一次 PR 尽量解决一个清晰问题。
- 不要提交 `node_modules/`、`dist/`、`output/`、录屏文件或本地 Cloudflare 状态。
- 不要提交密钥、令牌或个人环境配置。
- 如果修改了音频来源或处理链，请同步更新 `public/audio/README.md`。

提交贡献即表示你同意将自己的改动按本项目的 MIT License 发布。
